На оценку 8.

Программа copy_syscalls использует только системные вызовы для работы с файлами: open, read, write, close, fstat, chmod.
На вход подаются два аргумента командной строки — имя исходного файла и имя файла назначения.
Копирование содержимого реализовано с помощью циклического использования буфера фиксированного размера 32 байта: файл читается по 32 байта с помощью read() и немедленно записывается в выходной файл с помощью write().
Перед копированием программа получает права доступа исходного файла через fstat(). После завершения копирования программа устанавливает для файла назначения те же права доступа, что и у исходного файла, с помощью chmod(). Таким образом:

если исходный файл был исполняемым, то и новый файл остаётся исполняемым;

если исходный файл был обычным текстовым файлом без признака исполнимости, то новый файл также не становится исполняемым.

Запуск:
gcc -Wall -Wextra -std=c11 hw7_copy_syscalls.c -o copy_syscalls

./copy_syscalls input.txt output.txt
./copy_syscalls script.sh script_copy.sh
./copy_syscalls /bin/ls ls_copy
