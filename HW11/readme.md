# ДЗ 11
Программа измеряет максимально допустимую длину цепочки символьных ссылок именно для операции `open()`.
Работа ведётся в отдельной временной директории в `/tmp`, чтобы не засорять текущую папку.  
Цепочка строится так: создаётся файл `a`, затем `link_001 -> a`, `link_002 -> link_001`, ...
После создания каждого симлинка выполняется `open()`; если `open()` успешен - дескриптор сразу закрывается и цепочка продолжается.  
При первом провале печатается:
- глубина, на которой `open()` не сработал;
- максимальная глубина, при которой `open()` ещё был успешен;
- код ошибки (`errno`), обычно это `ELOOP`.

По умолчанию все созданные файлы и директория удаляются. Для отладки можно оставить их через `--keep`.

## Сборка
```bash
gcc -Wall -Wextra -O2 symlink_depth.c -o symlink_depth
````

## Запуск

Обычный режим (всё создаётся и затем очищается автоматически):

```bash
./symlink_depth
```

Оставить файлы после выполнения (удобно посмотреть цепочку):

```bash
./symlink_depth --keep
```

## Как протестировать

1. Запустить `./symlink_depth` и убедиться, что выводит рабочую директорию и итоговую глубину.
2. Запустить `./symlink_depth --keep`, затем проверить содержимое:

```bash
ls -la /tmp/symlink_depth.*
```

3. Удалить вручную (если запускал с `--keep`):

```bash
rm -rf /tmp/symlink_depth.<...>
```

## Bash-версия (опционально)

Почти то же самое, проверку делает через перенаправление:
`exec {fd}<"$name"` , после успеха fd сразу закрывается.

### Запуск

```bash
chmod +x symlink_depth.sh
./symlink_depth.sh
```

Оставить файлы после выполнения:

```bash
./symlink_depth.sh --keep
```
