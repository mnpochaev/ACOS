Реализованы две программы: sender и receiver, которые обмениваются пользовательскими сигналами. 

В sender после ввода PID собеседника и числа настраиваются обработчики SIGUSR1/SIGUSR2, которые используются как подтверждение приёма бита. Число приводится к типу uint32_t, после чего 32 бита по очереди вычисляются сдвигом и маской, для каждого бита выбирается соответствующий сигнал (SIGUSR1 для 0, SIGUSR2 для 1). Перед отправкой следующего бита sender ждёт подтверждение через глобальный флаг ack_received, который выставляется в обработчике сигнала. Для визуализации побитовой передачи добавил вызов usleep, чтобы биты появлялись на экране с задержкой.

В receiver настраиваются три обработчика: для SIGUSR1, SIGUSR2 и SIGINT. Обработчики битовых сигналов выполняют сдвиг накопленного значения и добавление 0/1, увеличивают счётчик принятых бит и отправляют обратно SIGUSR1 как ACK. Биты сразу выводятся в терминал с помощью write(STDOUT_FILENO, ...), чтобы избежать проблем с небезопасными в обработчиках функциями (printf). Обработчик SIGINT лишь выставляет флаг завершения, а в основном цикле программа выходит из pause(), после чего преобразует накопленное uint32_t к int32_t и выводит итоговое значение.
